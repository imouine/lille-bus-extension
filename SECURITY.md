# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please do not open a public issue immediately.

Instead, report it privately so it can be reviewed and fixed before public disclosure.

You can report security issues by contacting the maintainer through GitHub.

Please include as much information as possible:

- description of the vulnerability
- steps to reproduce the issue
- potential impact
- suggested fix (if available)

## Responsible Disclosure

Security issues will be investigated and addressed as quickly as possible.

Once a fix is available, the issue may be publicly documented in the repository.

## Scope

This policy applies to the source code contained in this repository.

Issues related to external services, APIs, or third-party infrastructure should be reported to their respective providers.
